Float详解（二）

2014 年8 月2 日 12:00 am

独立的float（一）

四、float的属性值：none、left、right

（1）none

float属性的默认值，只要用于设置元素对象不浮动，可以将浮动元素变为非浮动元素。

（2）left

使元素向左浮动，形成由左到右的一种流体。

（3）right

使元素向右浮动，形成由右到左的一种流体。

我觉得对于基础的讲解，w3cschool的例子是比较好的。

五、float之清除浮动

clear属性定义了元素的哪边上不允许出现浮动元素。

clear属性有四个值，分别为：none、left、right、both

概念的解释可以前往w3cschool

清除浮动主要有以下几种解决方案：

  • 使用内嵌样式：clear: [someone]

      <div class="wrap">
      <div class="left"></div>
      <p style="clear: left;"></p>
  </div>
	
  .wrap {
      background: #ddd;
  }
  .left {
      width: 400px;
      height: 200px;
      float: left;
      background: #00f;
  }

tip: 在html的结构里面出现控制样式的属性，这是不太好的做法，不太符合结构和样式分离的web标准。

  • 使用br标签

br标签有几个值，其实最开始我也不知道这个标签能够这样使用，查阅了资料了解。

br标签具有3个值：left、right、all，所有浏览器都支持用法和clear属性的值差不多，比如上面的例子可以写成：

<div class="wrap">
	<div class="left"></div>
	<br clear="left" />
</div>

.wrap {
	background: #ddd;
	zoom: 1; /* ie6 */
}

依然可以达到相同的效果，个人觉得br标签不到万不得已不使用，毕竟是带了一个换行的样式，但是有些html标签能够很节约css，并且可以很好的实现效果，所以灵活处理是必要的。

tip: 添加zoom属性是为了触发IE的HasLayout，这样才能够让div.wrap的高度不塌陷。

  • 使用display: table来模拟table清除浮动

这个方法我觉得可能使用得比较少，IE6不支持这属性的值，而且在ff和chrome这里面的实际效果和期望效果有差别。具体如下，改一下之前的html结构，来演示这种清除浮动的方式：

<div class="wrap">
	<div class="left">这是左边</div>
	<div class="right">这是右边</div>
</div>

.wrap {
	background: #ddd;
}
.left, .right {
	width: 400px;
	height: 200px;
}
.left {
	float: left;
	background: #00f;
}
.right {
	float: right;
	background: #eee;
}

以上例子未清除浮动的效果图：

将div.wrap的样式修改为：

.wrap {
	display: table;
	zoom: 1; /* ie6 */
	background: #ddd;
}

可以从上面两个效果比较，可以看出其实使用display这种方式是达不到预期的效果的。当然，也可以变通一下，其实这里有一个父容器的问题，我们可以为div.wrap定宽来看效果。

.wrap {
	display: table;
	zoom: 1;
	width: 900px;
	background: #fcc;
}

其实这样表面上解决了上面display后会紧密排列的问题，但是这样div.left和div.right就不会随窗口缩小而缩小他们中间的举例。

.wrap {
	display: table;
	zoom: 1;
	width: 900px;
	background: #fcc;
}

＝＝怎么感觉越写越多…先睡觉…

  • 使用overflow特殊清除浮动

这个方式介绍，按需求来采用方案。

按照之前给的例子，如果div.left和div.right均设置为float: left，那么会导致div.wrap外层元素的高度塌陷，可以利用overflow: hidden来让现代浏览器重新撑起高度，包裹住自己的子元素。

但是你会发现，其他浏览器都有了我们想要的效果，可是ie6会没什么变化，对呀，应该没有变化，我们使用overflow:hidden，并没有触发ie6撑起高度的条件，不能算是bug，个人觉得只是浏览器的实现不同。那我们要怎么解决呢？很简单，只需要触发ie6撑起高度的条件就行了，如果没什么特殊要求，使用zoom: 1来撑起ie6的塌陷高度是比较好的，不会影响其他元素的特性。所以总结如下：

.wrap { 
	overflow: hidden;
	zoom: 1; /* 你也可以加入ie6的hack标记 */
}

但是这样做也有不足的地方，overflow: hidden这个属性会吃掉超出元素的内容。

  • 使用伪类:after清除浮动

使用伪类的方式，其实在之前已经拿来使用过啦，现在再仔细研究下。大致写法如下：（当然也有另类一点的写法，比如用content: “\200B”这种使用字符来达到添加的字符内容本身不可见的效果）

.clearfix:after {
	display: block;
	content: "";
	height: 0;
	line-height: 0;
	visibility: hidden;
	clear: both;
}
.clearfix {
	zoom: 1; 
}

(1) display: block -> 这个好理解，先弄成块级元素

(2) content: “” -> 在.clearfix元素的末尾添加一个空元素，但这仅仅不够，所以会设置visibility

(3) visibility: hidden -> 让可见元素隐藏，这样就做到了真正忽略我们添加的content

(4) height: 0和line-height: 0 -> 这个也比较好理解，就是设置元素高度为0，看到不到，暂时没测试过是兼容哪一个浏览器，准备单独研究一下height（auto, 100%, 0, 18px等）特殊值对元素样式的影响。

(5) clear: both -> 清除浮动嘛，必要的。其实就是让浮动元素的盒子不影响到外层元素的盒子

(6) zoom: 1 -> 触发ie6的条件的，主要是让父元素的高度计算，按照自己和子元素的样式来计算

  • 总结

其实从上面的来看，就有两种：

(1) 使用clear属性带来的特性，直接清除浮动（得注意添加clear的位置，当然如果慢慢明白了原理就不用注意了，该是怎么回事就怎么用）

(2) 使用overflow和zoom等属性附带特定的值来触发元素撑起高度的条件，后来测试发现了好几种方案都是可行，所以追寻原理，后来发现，标准浏览器有自己的BFC，ie6／7有自己的HasLayout，其实不难理解，就是达到条件，让浏览器的解析换一种方式，不过也有比较细节的地方，需要好好总结。
