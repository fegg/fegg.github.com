DOM编程小结

Aug 2nd, 2014 12:00 am

一、节点层次

dom可以将任何html文档描述成一个多层节点的结构。在html中总共有12种节点类型，这些节点都继承自一个基类型。每种类型分别表示文档中不同的信息或者标记。每个节点都会拥有自己的特点、数据和方法，节点之间还有着相互关系。节点之间的各种关系构成了节点层次，这种层次可以用树形结构更加形象的表示。下面我们看一个简单的html代码：

<!DOCTYPE HTML>
<html>
<head>
    <meta charset="utf-8" />
    <title>节点层次</title>
</head>
<body>
    <p class="view-text">这里是一段文本。</p>
</body>
</html>

可以将以上这段代码表示一个简单的html文档层次结构。

［插图］

文档节点是每个文档的根节点，在改例子中，文档节点只有一个子节点，即：元素，我们称为文档元素。文档元素是文档的最外层元素，文档中的其它所有元素都包含在文档元素中，故：每个文档只有一个文档元素。

以下是常用的节点类型：

  1. 元素节点（Element Node）
  2. 文本节点（Text Node）

  3. 属性节点（Attribute Node）

     <p class="view-text">这里是一段文本。</p>

二、节点关系

1.父节点（parentNode）

2.子节点（childNode）

3.兄弟节点（siblingNode）

三、常用DOM操作属性和方法

  • 节点关系的属性操作
Attribute Note
parentNode 得到元素父节点对象，返回object
childNodes 得到元素所有的子节点，返回NodeList集合
children 得到元素所有的子HTML对象，返回HTMLCollection
previousSibling 得到元素前一个兄弟节点，返回object
nextSibling 得到元素的紧邻的下一个兄弟节点，返回object
firstChild 得到元素的第一个子结点
lastChild 得到元素的最后一个子节点
  • 节点本身的操作属性
Attribute Note
nodeType 元素的类型
nodeName 元素的名称
nodeValue 元素的值
  • 常用DOM操作方法
Method Note
getElementById 通过id获取元素
getElementsByTagName 通过节点名称获取元素
getAttribute 取得元素的属性值
setAttribute 取得元素的属性值
appendChild 向元素的末尾追加文档内容
insertBefore 向元素的前面插入文档内容
createElement 创建元素节点
createTextNode 创建文本节点
